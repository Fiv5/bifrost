# www.qq.com 转发规则测试
# 测试目标:
# 1. /api/ 路径 - 直接转发到原始服务 (不做特殊处理)
# 2. 其他 HTTPS 请求 - 转发到 http://localhost:8000/
# 3. WSS 请求 - 转发到 ws://localhost:8000/

# 规则1: /api/ 路径排除，让其直接访问原始服务
# 使用 excludeFilter 排除 /api/ 路径，其他路径显式转发到本地 HTTP 服务。
# 这里使用 http:// 前缀，确保 HTTPS->HTTP 重写会触发 TLS interception。
www.qq.com http://127.0.0.1:8000/ excludeFilter:///api/

# 规则2: WebSocket 请求转发到本地 WS 服务
# 使用 wss:// 前缀明确指定协议匹配，排除/api/路径
wss://www.qq.com/ ws://127.0.0.1:8000 excludeFilter:///api/

# 测试用例:
# 1. curl -x http://127.0.0.1:9900 https://www.qq.com/
#    预期: 转发到 127.0.0.1:8000
# 2. curl -x http://127.0.0.1:9900 https://www.qq.com/api/test
#    预期: 直接访问原始服务 (被 excludeFilter 排除)
# 3. websocat -v wss://www.qq.com/ --proxy http://127.0.0.1:9900
#    预期: 转发到 ws://127.0.0.1:8000
