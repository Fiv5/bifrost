# nextoncall.bytedance.net 转发规则测试
# 测试目标:
# 1. /api/ 路径 - 直接转发到原始服务 (不做特殊处理)
# 2. 其他 HTTPS 请求 - 转发到 http://localhost:8000/
# 3. WSS 请求 - 转发到 ws://localhost:8000/

# 规则1: /api/ 路径排除，让其直接访问原始服务
# 使用 excludeFilter 排除 /api/ 路径，其他路径转发到本地服务
nextoncall.bytedance.net host://127.0.0.1:8000 excludeFilter:///api/

# 规则2: WebSocket 请求转发到本地 WS 服务
# 使用 wss:// 前缀明确指定协议匹配，排除/api/路径
wss://nextoncall.bytedance.net/ ws://127.0.0.1:8000 excludeFilter:///api/

# 测试用例:
# 1. curl -x http://127.0.0.1:9900 https://nextoncall.bytedance.net/
#    预期: 转发到 127.0.0.1:8000
# 2. curl -x http://127.0.0.1:9900 https://nextoncall.bytedance.net/api/test
#    预期: 直接访问原始服务 (被 excludeFilter 排除)
# 3. websocat -v wss://nextoncall.bytedance.net/ --proxy http://127.0.0.1:9900
#    预期: 转发到 ws://127.0.0.1:8000
