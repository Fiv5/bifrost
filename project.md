# Bifrost 项目架构文档

## 项目概述

Bifrost 是一个用 Rust 编写的高性能 HTTP/HTTPS/SOCKS5 代理服务器，是 whistle 的 Rust 重写版本。它提供强大的请求拦截、修改和规则配置能力，支持 TLS 解密、插件扩展等高级功能。

## 模块说明

### bifrost-core (核心库)

核心库提供基础功能，是整个项目的基石。

#### 主要组件

| 组件       | 路径                           | 功能描述                         |

| 协议定义   | `src/protocol.rs`              | 定义 78 种协议操作类型           |
| 规则解析   | `src/rule/parser.rs`           | 解析规则语法，支持多种格式       |
| 规则类型   | `src/rule/types.rs`            | 定义 Rule 核心数据结构           |
| 规则上下文 | `src/rule/context.rs`          | 请求匹配上下文                   |

| 规则组     | `src/rule/group.rs`            | 规则分组管理                     |
| 规则解析器 | `src/rule/resolver.rs`         | 规则匹配和解析                   |
| 模板处理   | `src/rule/template.rs`         | 变量模板展开                     |
| 值存储     | `src/rule/value_store.rs`      | Values 变量管理                  |
| 值来源     | `src/rule/value_source.rs`     | 值来源类型解析                   |

| 域名匹配   | `src/matcher/domain.rs`        | 精确域名匹配                     |
| 通配符匹配 | `src/matcher/wildcard.rs`      | 通配符模式匹配                   |
| 路径通配符 | `src/matcher/path_wildcard.rs` | 路径通配符匹配 (^)               |

### bifrost-proxy (代理服务器)

代理服务器核心实现，处理所有网络请求。

#### 主要组件

| 组件 | 路径 | 功能描述 |
|------|------|----------|
| 服务器入口 | `src/server.rs` | 代理服务器主逻辑 |
| HTTP 处理 | `src/http.rs` | HTTP 请求处理 |
| SOCKS5 代理 | `src/socks.rs` | SOCKS5 协议支持 |
| 隧道处理 | `src/tunnel.rs` | CONNECT 隧道和 TLS 拦截 |
| WebSocket | `src/websocket.rs` | WebSocket 代理 |
| 请求处理 | `src/request.rs` | 请求修改和转发 |
| 响应处理 | `src/response.rs` | 响应修改和返回 |
| Mock | `src/mock.rs` | 模拟响应生成 |
| Body | `src/body.rs` | 请求/响应体处理 |
| 协议检测 | `src/protocol/detect.rs` | 协议自动检测 |
| HTTP/1 | `src/protocol/http1.rs` | HTTP/1.x 处理 |
| HTTP/2 | `src/protocol/h2.rs` | HTTP/2 处理 |
| 连接池 | `src/protocol/pool.rs` | HTTP 连接池 |
| SSE | `src/protocol/sse.rs` | Server-Sent Events |
| 流处理 | `src/protocol/stream.rs` | 流式传输 |

## 规则解析流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           规则解析流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 输入: "*.example.com host://127.0.0.1 includeFilter://api"          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. 词法分析 (RuleParser)                                            │   │
│  │    - 识别 pattern: *.example.com                                    │   │
│  │    - 识别 protocol: host                                            │   │
│  │    - 识别 value: 127.0.0.1                                          │   │
│  │    - 识别 filter: includeFilter://api                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 2. 模式类型检测 (detect_pattern_type)                               │   │
│  │    - *.example.com → Wildcard                                       │   │
│  │    - example.com → Domain                                           │   │
│  │    - 192.168.1.1 → IP                                               │   │
│  │    - /regex/ → Regex                                                │   │
│  │    - ^example.com/* → PathWildcard                                  │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 3. 创建匹配器 (parse_pattern)                                       │   │
│  │    - WildcardMatcher::new("*.example.com")                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. 协议解析 (Protocol::parse)                                       │   │
│  │    - "host" → Protocol::Host                                        │   │
│  │    - 处理别名 (hosts → host, skip → ignore)                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 5. 值来源解析 (ValueSource::parse)                                  │   │
│  │    - "127.0.0.1" → Inline                                           │   │
│  │    - "/path/to/file" → FilePath                                     │   │
│  │    - "{varName}" → ValueRef                                         │   │
│  │    - "(content)" → ParenContent                                     │   │
│  │    - "http://..." → RemoteUrl                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 6. 过滤器解析                                                       │   │
│  │    - includeFilter://api → Filter { pattern: "api", ... }           │   │
│  │    - excludeFilter://test → Filter { pattern: "test", ... }         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 7. 构建 Rule 对象                                                   │   │
│  │    Rule {                                                           │   │
│  │        pattern: "*.example.com",                                    │   │
│  │        matcher: WildcardMatcher,                                    │   │
│  │        protocol: Host,                                              │   │
│  │        value: "127.0.0.1",                                          │   │
│  │        value_source: Inline,                                        │   │
│  │        include_filters: [Filter("api")],                            │   │
│  │        ...                                                          │   │
│  │    }                                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 规则匹配流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           规则匹配流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────┐     │
│  │ 请求: GET https://api.example.com/users?id=123                    │     │
│  │       Host: api.example.com                                       │     │
│  │       Headers: { "Content-Type": "application/json" }             │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. 构建匹配上下文 (RuleContext)                                     │   │
│  │    - url: "https://api.example.com/users?id=123"                    │   │
│  │    - host: "api.example.com"                                        │   │
│  │    - path: "/users"                                                 │   │
│  │    - query: "id=123"                                                │   │
│  │    - method: "GET"                                                  │   │
│  │    - headers: {...}                                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 2. 遍历规则组 (按优先级排序)                                        │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ 规则列表 (已排序):                                           ││   │
│  │    │   [1] api.example.com host://... (priority: 100)             ││   │
│  │    │   [2] 192.168.1.1 proxy://...    (priority: 95)              ││   │
│  │    │   [3] /api\..*/ resHeaders://... (priority: 80)              ││   │
│  │    │   [4] *.example.com host://...   (priority: 55)              ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 3. 对每条规则执行匹配                                               │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ matcher.matches(url, host, path)                             ││   │
│  │    │   ├── DomainMatcher: 精确比较 host                           ││   │
│  │    │   ├── WildcardMatcher: 通配符匹配                            ││   │
│  │    │   ├── RegexMatcher: 正则匹配 url                             ││   │
│  │    │   ├── IpMatcher: IP/CIDR 比较                                ││   │
│  │    │   └── PathWildcardMatcher: 路径匹配                          ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. 检查过滤器                                                       │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ includeFilter 检查:                                          ││   │
│  │    │   - URL/Path 包含过滤关键字?                                 ││   │
│  │    │   - Headers 包含过滤关键字?                                  ││   │
│  │    │   - Body 包含过滤关键字?                                     ││   │
│  │    │                                                              ││   │
│  │    │ excludeFilter 检查:                                          ││   │
│  │    │   - 匹配排除条件则跳过此规则                                 ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 5. 处理 multi-match 协议                                            │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ 单匹配协议 (host, proxy, method, ...):                       ││   │
│  │    │   - 首次匹配即返回                                           ││   │
│  │    │                                                              ││   │
│  │    │ 多匹配协议 (reqHeaders, resHeaders, resBody, ...):           ││   │
│  │    │   - 收集所有匹配规则                                         ││   │
│  │    │   - 合并应用                                                 ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 6. 返回匹配结果                                                     │   │
│  │    ResolvedRules {                                                  │   │
│  │        host_rule: Some(Rule { ... }),                               │   │
│  │        req_headers: vec![Rule, Rule, ...],                          │   │
│  │        res_headers: vec![Rule, Rule, ...],                          │   │
│  │        ...                                                          │   │
│  │    }                                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 开发指南

### 添加新协议

1. 在 `bifrost-core/src/protocol.rs` 中添加新枚举值
2. 实现 `parse()` 和 `to_str()` 方法
3. 在 `category()` 中指定协议分类
4. 如需多匹配支持，添加到 `MULTI_MATCH_PROTOCOLS`

### 添加新匹配器

1. 在 `bifrost-core/src/matcher/` 下创建新文件
2. 实现 `Matcher` trait
3. 在 `factory.rs` 中添加检测和解析逻辑

### 添加新插件 Hook

1. 在 `bifrost-plugin/src/hook.rs` 中添加新枚举值
2. 更新 `PluginHook::ALL` 数组
3. 实现 `as_str()` 和 `parse()` 方法
