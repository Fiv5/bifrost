# Bifrost 项目架构文档

## 项目概述

Bifrost 是一个用 Rust 编写的高性能 HTTP/HTTPS/SOCKS5 代理服务器，是 whistle 的 Rust 重写版本。它提供强大的请求拦截、修改和规则配置能力，支持 TLS 解密、插件扩展等高级功能。

### 1. HTTP/HTTPS 代理模式

HTTP/HTTPS 代理是最常用的代理模式，支持普通 HTTP 请求转发和 HTTPS 隧道/拦截。

#### 核心组件

| 组件        | 文件路径                  | 功能描述                     |
| ----------- | ------------------------- | ---------------------------- |
| HTTP 处理器 | `proxy/http/handler.rs`   | HTTP 请求/响应处理、规则应用 |
| HTTPS 隧道  | `proxy/http/tunnel.rs`    | CONNECT 隧道、TLS MITM 拦截  |
| WebSocket   | `proxy/http/websocket.rs` | WebSocket 协议代理           |

### 2. SOCKS5 TCP 代理模式

SOCKS5 TCP 代理提供通用的 TCP 隧道功能，支持 CONNECT 命令建立 TCP 连接。

#### 核心组件

| 组件          | 文件路径                          | 功能描述                  |
| ------------- | --------------------------------- | ------------------------- |
| SOCKS5 服务器 | `proxy/socks/tcp.rs`              | SOCKS5 协议处理、握手认证 |
| SOCKS5 处理器 | `proxy/socks/tcp.rs:SocksHandler` | 单个连接处理逻辑          |

#### 地址类型 (ATYP)

| 值   | 类型 | 格式              |
| ---- | ---- | ----------------- |
| 0x01 | IPv4 | 4 字节地址        |
| 0x03 | 域名 | 1 字节长度 + 域名 |
| 0x04 | IPv6 | 16 字节地址       |

#### SOCKS5 命令类型

| 命令          | 值   | 说明              |
| ------------- | ---- | ----------------- |
| CONNECT       | 0x01 | TCP 隧道连接      |
| BIND          | 0x02 | TCP 绑定 (不支持) |
| UDP ASSOCIATE | 0x03 | UDP 关联          |

#### 关键函数

| 函数名                                    | 功能描述                        |
| ----------------------------------------- | ------------------------------- |
| `SocksServer::run`                        | 启动 SOCKS5 服务器              |
| `SocksHandler::handle_handshake`          | 处理握手认证                    |
| `SocksHandler::handle_request`            | 处理 CONNECT/UDP ASSOCIATE 请求 |
| `SocksHandler::connect_and_relay`         | TCP 连接并中继数据              |
| `SocksHandler::relay_with_tls_intercept`  | TLS 拦截模式中继                |
| `SocksHandler::relay_with_http_intercept` | HTTP 拦截模式中继               |

### 3. SOCKS5 UDP 代理模式

SOCKS5 UDP 代理通过 UDP ASSOCIATE 命令建立 UDP 数据报转发通道。

#### 核心组件

| 会话管理 | `proxy/socks/udp.rs:UdpSession` | UDP 会话状态管理 |

#### UDP 数据报头格式

```
+----+------+------+----------+----------+----------+
|RSV | FRAG | ATYP | DST.ADDR | DST.PORT |   DATA   |
+----+------+------+----------+----------+----------+
| 2  |  1   |  1   | Variable |    2     | Variable |
+----+------+------+----------+----------+----------+

RSV: 保留字段 (0x0000)
FRAG: 分片编号 (0x00 = 不分片)
ATYP: 地址类型 (同 TCP)
DST.ADDR: 目标地址
DST.PORT: 目标端口
DATA: 实际 UDP 数据负载
```

### 4. HTTP/3 + MASQUE 代理模式

HTTP/3 代理基于 QUIC 协议，支持 MASQUE (Multiplexed Application Substrate over QUIC Encryption) 标准，提供现代化的代理能力。

#### 核心组件

| 组件          | 文件路径               | 功能描述                   |
| ------------- | ---------------------- | -------------------------- |
| HTTP/3 服务器 | `http3/server.rs`      | QUIC/HTTP/3 服务器         |
| CONNECT-UDP   | `http3/connect_udp.rs` | UDP over HTTP/3 (RFC 9298) |
| Capsule 协议  | `http3/capsule.rs`     | Capsule 封装协议           |
| HTTP/3 隧道   | `http3/tunnel.rs`      | TCP over QUIC 隧道         |
| HTTP/3 代理   | `http3/proxy.rs`       | HTTP/3 请求代理            |

## 规则解析流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           规则解析流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 输入: "*.example.com host://127.0.0.1 includeFilter://api"          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. 词法分析 (RuleParser)                                            │   │
│  │    - 识别 pattern: *.example.com                                    │   │
│  │    - 识别 protocol: host                                            │   │
│  │    - 识别 value: 127.0.0.1                                          │   │
│  │    - 识别 filter: includeFilter://api                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 2. 模式类型检测 (detect_pattern_type)                               │   │
│  │    - *.example.com → Wildcard                                       │   │
│  │    - example.com → Domain                                           │   │
│  │    - 192.168.1.1 → IP                                               │   │
│  │    - /regex/ → Regex                                                │   │
│  │    - ^example.com/* → PathWildcard                                  │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 3. 创建匹配器 (parse_pattern)                                       │   │
│  │    - WildcardMatcher::new("*.example.com")                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. 协议解析 (Protocol::parse)                                       │   │
│  │    - "host" → Protocol::Host                                        │   │
│  │    - 处理别名 (hosts → host, skip → ignore)                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 5. 值来源解析 (ValueSource::parse)                                  │   │
│  │    - "127.0.0.1" → Inline                                           │   │
│  │    - "/path/to/file" → FilePath                                     │   │
│  │    - "{varName}" → ValueRef                                         │   │
│  │    - "(content)" → ParenContent                                     │   │
│  │    - "http://..." → RemoteUrl                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 6. 过滤器解析                                                       │   │
│  │    - includeFilter://api → Filter { pattern: "api", ... }           │   │
│  │    - excludeFilter://test → Filter { pattern: "test", ... }         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 7. 构建 Rule 对象                                                   │   │
│  │    Rule {                                                           │   │
│  │        pattern: "*.example.com",                                    │   │
│  │        matcher: WildcardMatcher,                                    │   │
│  │        protocol: Host,                                              │   │
│  │        value: "127.0.0.1",                                          │   │
│  │        value_source: Inline,                                        │   │
│  │        include_filters: [Filter("api")],                            │   │
│  │        ...                                                          │   │
│  │    }                                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 规则匹配流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           规则匹配流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌───────────────────────────────────────────────────────────────────┐     │
│  │ 请求: GET https://api.example.com/users?id=123                    │     │
│  │       Host: api.example.com                                       │     │
│  │       Headers: { "Content-Type": "application/json" }             │     │
│  └───────────────────────────────────────────────────────────────────┘     │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. 构建匹配上下文 (RuleContext)                                     │   │
│  │    - url: "https://api.example.com/users?id=123"                    │   │
│  │    - host: "api.example.com"                                        │   │
│  │    - path: "/users"                                                 │   │
│  │    - query: "id=123"                                                │   │
│  │    - method: "GET"                                                  │   │
│  │    - headers: {...}                                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 2. 遍历规则组 (按优先级排序)                                        │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ 规则列表 (已排序):                                           ││   │
│  │    │   [1] api.example.com host://... (priority: 100)             ││   │
│  │    │   [2] 192.168.1.1 proxy://...    (priority: 95)              ││   │
│  │    │   [3] /api\..*/ resHeaders://... (priority: 80)              ││   │
│  │    │   [4] *.example.com host://...   (priority: 55)              ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 3. 对每条规则执行匹配                                               │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ matcher.matches(url, host, path)                             ││   │
│  │    │   ├── DomainMatcher: 精确比较 host                           ││   │
│  │    │   ├── WildcardMatcher: 通配符匹配                            ││   │
│  │    │   ├── RegexMatcher: 正则匹配 url                             ││   │
│  │    │   ├── IpMatcher: IP/CIDR 比较                                ││   │
│  │    │   └── PathWildcardMatcher: 路径匹配                          ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 4. 检查过滤器                                                       │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ includeFilter 检查:                                          ││   │
│  │    │   - URL/Path 包含过滤关键字?                                 ││   │
│  │    │   - Headers 包含过滤关键字?                                  ││   │
│  │    │   - Body 包含过滤关键字?                                     ││   │
│  │    │                                                              ││   │
│  │    │ excludeFilter 检查:                                          ││   │
│  │    │   - 匹配排除条件则跳过此规则                                 ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 5. 处理 multi-match 协议                                            │   │
│  │    ┌──────────────────────────────────────────────────────────────┐│   │
│  │    │ 单匹配协议 (host, proxy, method, ...):                       ││   │
│  │    │   - 首次匹配即返回                                           ││   │
│  │    │                                                              ││   │
│  │    │ 多匹配协议 (reqHeaders, resHeaders, resBody, ...):           ││   │
│  │    │   - 收集所有匹配规则                                         ││   │
│  │    │   - 合并应用                                                 ││   │
│  │    └──────────────────────────────────────────────────────────────┘│   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 6. 返回匹配结果                                                     │   │
│  │    ResolvedRules {                                                  │   │
│  │        host_rule: Some(Rule { ... }),                               │   │
│  │        req_headers: vec![Rule, Rule, ...],                          │   │
│  │        res_headers: vec![Rule, Rule, ...],                          │   │
│  │        ...                                                          │   │
│  │    }                                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 开发指南

### 添加新协议

1. 在 `bifrost-core/src/protocol.rs` 中添加新枚举值
2. 实现 `parse()` 和 `to_str()` 方法
3. 在 `category()` 中指定协议分类
4. 如需多匹配支持，添加到 `MULTI_MATCH_PROTOCOLS`

### 添加新匹配器

1. 在 `bifrost-core/src/matcher/` 下创建新文件
2. 实现 `Matcher` trait
3. 在 `factory.rs` 中添加检测和解析逻辑
