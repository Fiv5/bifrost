# P-03: 规则顺序优先级测试
# 需求: 对于修改类操作符(resHeaders等)，后定义的规则会覆盖前面的

# 第一条规则 (会被后面的覆盖)
order-test.local 127.0.0.1:3000 resHeaders://{match_first}

# 第二条规则 (会被后面的覆盖)
order-test.local 127.0.0.1:3000 resHeaders://{match_second}

# 第三条规则 (最终生效)
order-test.local 127.0.0.1:3000 resHeaders://{match_third}

# 测试用例:
# 1. 请求 http://order-test.local/
#    响应应包含 X-Match: third (最后一条规则生效)
# 2. 前面的规则被后面的覆盖

```match_first
X-Match: first
```

```match_second
X-Match: second
```

```match_third
X-Match: third
```
