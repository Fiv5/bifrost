## 真实人类测试

区别于单元测试（mock 环境 + 断言）和端到端脚本测试（自动化脚本 + mock 服务），真实人类测试的核心思路是：**使用真实的远程服务和真实的用户数据，以人类操作者的视角逐步执行每一个命令，观察真实输出，验证功能正确性。**

### 核心理念

1. **使用默认数据目录**：不隔离数据目录，直接使用用户的 `~/.bifrost` 数据（含已登录的 token、已同步的配置等），模拟真实用户环境
2. **连接真实远程服务**：通过已登录账号连接真实的同步服务，操作真实的 group、rule 等数据
3. **逐步执行 + 人工观察**：每执行一条命令后检查输出、验证状态变更，而非批量断言
4. **闭环验证**：不仅验证 CLI 输出正确，还验证功能是否真正生效（如启用规则后通过代理发请求验证转发行为）

### 适用场景

- CLI 子命令的完整 CRUD 流程验证
- 涉及远程 API 交互的功能（group/sync/upgrade 等需要登录态的命令）
- 代理规则的热加载/卸载行为验证
- 分页、搜索等交互功能的边界验证
- JSON 反序列化兼容性验证（远程 API 返回的数据可能包含 null、类型不一致等边缘情况）

### 执行方法

#### 第一步：启动服务（使用默认数据目录）

```bash
cargo run --bin bifrost -- start -p 8800 --unsafe-ssl
```

注意：使用默认数据目录（`~/.bifrost`），确保已完成登录（`bifrost login`）。端口避免使用 9900（正式环境端口）。

#### 第二步：逐步执行命令并验证

以 `group` 子命令为例，完整的测试路径：

```bash
# 1. 列表（基础）
cargo run --bin bifrost -- -p 8800 group list

# 2. 列表（分页）
cargo run --bin bifrost -- -p 8800 group list --offset 50 --limit 5

# 3. 列表（最后一页 — 验证无"下一页"提示）
cargo run --bin bifrost -- -p 8800 group list --offset 170 --limit 50

# 4. 列表（关键词搜索）
cargo run --bin bifrost -- -p 8800 group list -k Bifrost

# 5. 查看详情
cargo run --bin bifrost -- -p 8800 group show <group_id>

# 6. 规则列表
cargo run --bin bifrost -- -p 8800 group rule list <group_id>

# 7. 添加规则
cargo run --bin bifrost -- -p 8800 group rule add <group_id> test-rule -c "httpbin.org host://127.0.0.1:9999"

# 8. 查看规则
cargo run --bin bifrost -- -p 8800 group rule show <group_id> test-rule

# 9. 更新规则
cargo run --bin bifrost -- -p 8800 group rule update <group_id> test-rule -c "httpbin.org host://127.0.0.1:8888
api.example.com host://127.0.0.1:7777"

# 10. 验证更新生效（再次 show 确认内容变更）
cargo run --bin bifrost -- -p 8800 group rule show <group_id> test-rule

# 11. 启用规则
cargo run --bin bifrost -- -p 8800 group rule enable <group_id> test-rule

# 12. 验证代理规则生效（应返回 502，因为目标端口无服务）
curl -s -o /dev/null -w "HTTP %{http_code}" --proxy http://127.0.0.1:8800 http://httpbin.org/ --max-time 3

# 13. 禁用规则
cargo run --bin bifrost -- -p 8800 group rule disable <group_id> test-rule

# 14. 验证代理规则已卸载（应返回 200，正常访问）
curl -s -o /dev/null -w "HTTP %{http_code}" --proxy http://127.0.0.1:8800 http://httpbin.org/get --max-time 5

# 15. 删除规则
cargo run --bin bifrost -- -p 8800 group rule delete <group_id> test-rule

# 16. 验证删除（rule list 确认已移除）
cargo run --bin bifrost -- -p 8800 group rule list <group_id>
```

#### 第三步：关键验证点

每一步执行后需要检查：

| 检查项 | 方法 |
|--------|------|
| 命令退出码 | 必须为 0 |
| 输出格式 | 表格/详情是否排版正确、字段完整 |
| 数据一致性 | add 后 list 能看到、delete 后 list 不再包含 |
| 代理行为 | enable 后 curl 通过代理应走规则转发；disable 后恢复直连 |
| 分页正确性 | Page X/Y 计算正确、最后一页无"下一页"提示 |
| 空值/null 兼容 | 远程 API 返回 null 字段不导致解析崩溃 |

### 与其他测试方式的对比

| 维度 | 单元测试 | E2E 脚本测试 | 真实人类测试 |
|------|----------|-------------|-------------|
| 数据来源 | Mock JSON | Mock 服务 | 真实远程 API |
| 环境 | 隔离 | 临时目录 | 默认用户目录 |
| 执行方式 | 自动断言 | 自动化脚本 | 逐步手动执行 |
| 发现的问题类型 | 逻辑错误 | 集成问题 | 数据兼容性、边缘情况、用户体验 |
| 典型案例 | 解析逻辑测试 | 规则转发验证 | 远程 API 返回 null 字段导致解析失败 |

### 注意事项

- 测试操作会影响真实数据（如添加/删除 group rule），测试完毕后务必清理测试数据
- 不要修改其他用户可见的公共 group 规则，选择自己有写权限的 private group 或测试 group
- 如果远程服务不可用（网络问题、未登录等），命令应给出清晰错误提示而非 panic
- 停止测试服务后无需清理数据目录（使用的是默认目录）
