## 快速构建启动（推荐）

目标：仅构建后端并快速启动代理服务（可跳过前端构建）。

首先需要复制`~/.bifrost/cert/client.crt`和`~/.bifrost/cert/client.key`到启动目录，确保启动时可以加载到客户端证书。例如`.bifrost-e2e-test`目录。

```bash
# 仅构建后端并快速启动（跳过前端构建）
SKIP_FRONTEND_BUILD=1 cargo run --bin bifrost -- start -p 8890 --unsafe-ssl

# 或使用 Makefile 快速启动
make dev
```

等待构建完成，并启动服务

使用curl测试
```bash
# 测试代理是否正常工作
curl -v http://127.0.0.1:8890/
```

## 前端 UI 快速启动与验证（dev 模式）

目标：本地快速验证管理端 UI；前端 devserver 会将 `/_bifrost/api` 与 `/_bifrost/ws` 代理到目标代理服务端口。

```bash
# 启动代理（示例端口 8890）
export BIFROST_DATA_DIR=./.bifrost-e2e-test
rm -rf "$BIFROST_DATA_DIR"
SKIP_FRONTEND_BUILD=1 cargo run --bin bifrost -- start -p 8890 --unsafe-ssl --skip-cert-check

# 启动前端 devserver，并将代理目标指向 8890
pnpm -C web dev -- --backend-port 8890
```

验证：

1. 打开：`http://127.0.0.1:3000/_bifrost/`
2. 确认页面可正常加载、列表有数据/可操作
3. 如需确认代理是否生效，可打开浏览器 Network，检查请求是否走 `/_bifrost/api` 路径且返回正常
